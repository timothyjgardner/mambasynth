# MambaSynth Research Log

Record of experiments, architectural decisions, and findings from this project exploring causal Mamba models for representation learning on synthetic time series.

## Project Goal

Learn **structured, cluster-separable representations** from synthetic Markov-switching time series using causal sequence models. The data consists of 10 overlapping oscillatory circles in 20 dimensions, with Markov-governed state transitions. A good model should build internal representations where time steps belonging to the same circle cluster together — measured by **silhouette score** on UMAP embeddings.

## Data

Generated by `markov_circles_timeseries.py` (from the `synthetic-jepa` project, copied to `data/`).

- **Dimensions**: 20D observations
- **States**: 10 Markov-switching circles with overlapping orbits
- **Noise levels tested**: 1× (noise_std=2.83), 2× (noise_std=5.66)
- **Drift**: With and without random-walk drift on circle centers
- **Total length**: 100,000 time steps

The random-walk drift makes the task harder because circle centers move over time, so the model can't rely on absolute position — it must learn the oscillatory dynamics.

## Phase 1: Supervised Next-Step Prediction

**Script**: `masked_model_gpu_mamba_next.py`

Standard autoregressive model: predict x[t+1] from x[1..t] in observation space. 7-layer Mamba encoder with output head.

**Key result**: Peak silhouette **0.435** (Layer 6) at 200 epochs on noisy random-walk data. Representations improve monotonically through layers, with intrinsic dimension dropping from ~12D (input) to ~4D (deep layers).

### Reservoir Computing Ablation

Freezing Mamba's input-dependent gating (x_proj, dt_proj) and state transition matrix (A_log) turns the model into a deep liquid state machine — 7 stacked random reservoirs with learned projections between them.

**Finding**: The frozen model learns representations qualitatively comparable to the full model. This suggests Mamba's selectivity is not critical for this task — the random SSM provides a sufficiently rich bank of temporal mixing patterns (57,344 independent temporal modes across 7 layers).

## Phase 2: Mamba-JEPA (Self-Supervised)

**Script**: `mamba_jepa.py`

Joint Embedding Predictive Architecture: predict the *target encoder's representation* of x[t+1] from the context encoder's representation of x[t]. The target encoder is an EMA copy of the context encoder (no gradients). Since Mamba is inherently causal, no masking is needed.

### Predictor Capacity Controls Representation Quality

The most important finding of the JEPA experiments:

| Predictor | Val JEPA Loss | Peak Silhouette | Best Layer |
|-----------|---------------|-----------------|------------|
| 2-layer Mamba | 0.0191 | 0.381 | Layer 7 |
| **Per-position MLP** | **0.0470** | **0.547** | **Layer 8 (norm)** |

A weaker MLP predictor (no temporal modeling, just position-wise) forces the encoder to build richer representations because the predictor can't compensate by learning temporal dependencies itself. The Mamba predictor achieves lower JEPA loss but *worse* downstream representations — the encoder becomes "lazy."

### Sequence Length Effects

| seq_len | Val JEPA Loss | Peak Silhouette | Best Layer |
|---------|---------------|-----------------|------------|
| 128 | 0.0293 | 0.331 | Layer 7 |
| 512 | 0.0420 | 0.547 | Layer 8 |
| 1024 | 0.0420 | **0.599** | Layer 8 |

Longer windows give the encoder more of each circle's trajectory per window. `seq_len=1024` achieves the best result on noisy random-walk data: **0.599** silhouette.

### JEPA vs Supervised

MLP-predictor JEPA at 1000 epochs (silhouette 0.547) surpasses supervised next-step prediction at 200 epochs (0.435) by a wide margin. The JEPA objective — predicting in representation space rather than observation space — encourages more abstract, state-discriminative features.

### Training Duration: Diminishing Returns

| Epochs | Peak Silhouette | Best Layer | Notes |
|--------|-----------------|------------|-------|
| 200 | 0.453 | Layer 8 | Early training |
| 1000 | **0.547** | Layer 8 | Sweet spot |
| 3000 | 0.394 | Layer 2 | Overspecialized |

Training for 3000 epochs *degraded* representations. The deeper layers became overly temporally ordered ("stringy" UMAP manifolds) rather than maintaining discrete cluster identity. The sweet spot was around 1000 epochs.

### Deep Supervision (Per-Layer Loss)

**Flag**: `--per-layer-loss`

Each encoder layer (including final norm) gets its own MLP predictor head. The total loss is the mean of per-layer MSEs.

**Stability issue**: Intermediate layer outputs initially lacked normalization, causing loss explosion. Fixed by adding per-layer `LayerNorm` modules in `MambaEncoder`.

**Result**: Peak silhouette 0.283 on random-walk data — worse than standard JEPA (0.547). Deep supervision may spread learning capacity too thinly, preventing deeper layers from specializing.

## Phase 3: Multi-Scale Architecture

**Originally**: `mamba_jepa_multiscale.py` (separate script)
**Now**: Integrated into `mamba_jepa.py` via `--strides` flag

### Motivation

The baseline Mamba implicitly learns multiple timescales through its SSM state, but there's no explicit architectural bias toward timescale separation. A convolutional front-end with strided downsampling physically forces different branches to operate at different temporal resolutions.

### Architecture

```
Single branch (stride=4):
  Input (B, T, D_in)
    → CausalConv1d(kernel=8, stride=4) → (B, T/4, D)
    → 7 Mamba layers (operating at 4× slower clock)
    → LayerNorm → upsample (repeat-interleave) → (B, T, D)

Multi-branch (strides=[1, 4, 16]):
  Input (B, T, D_in)
    ├→ Branch 1: Linear         → 7 Mamba layers → (B, T, D)
    ├→ Branch 2: Conv(stride=4) → 7 Mamba layers → upsample → (B, T, D)
    └→ Branch 3: Conv(stride=16)→ 7 Mamba layers → upsample → (B, T, D)
  Concatenate → Linear(3D→D) + LayerNorm → (B, T, D)
```

The stride-16 branch physically can't represent anything faster than 16-step fluctuations. Its Mamba layers are forced to model slow structure (circle identity, orbital phase).

### Timescale Design Choices Considered

1. **Fixed geometric strides** (chosen): Powers of a base (e.g., 1, 4, 16) — simple, no tuning
2. **Matched to data**: Analyze data autocorrelation and set strides accordingly — requires domain knowledge
3. **Learnable lags / decay rates**: Let the model discover timescales — increases optimization difficulty

### Results on 2× Noise Data (noise_std=5.66, no random walk)

**Single branch, stride=4** (best multi-scale result):

| Epochs | Peak Silhouette | Val JEPA Loss |
|--------|-----------------|---------------|
| 500 | 0.359 | — |
| **1000** | **0.515** | 0.0778 |

Comparable to the baseline JEPA (0.547 on 1× noise), showing the conv head works well even on harder data.

**Single branch, stride=16**: Negative silhouette. Mamba sees only 32-step sequences (512/16), losing too much temporal information. The representations become unstructured.

**Multi-branch, strides=[1, 4, 16]**: Very low validation loss but **poor silhouette** despite good loss. Diagnosed as a **"lazy branch" problem**: the stride-1 branch can easily predict the next step from local information, making slower branches redundant. The fusion layer learns to prioritize the fast branch, and the slow branches contribute little to the final representation.

### Lazy Branch Problem

With a next-step prediction loss, the stride-1 branch dominates because:
- It has full temporal resolution
- Next-step prediction is mostly a local task
- The fusion layer (Linear + LayerNorm) has no incentive to use slow-branch features

This is a fundamental tension: multi-scale *architecture* combined with a single-step *loss* doesn't force the model to use all scales. Potential solutions (not yet tested):
- Multi-horizon loss (predict t+1, t+4, t+16)
- Drop the stride-1 branch entirely (force slow processing)
- Separate losses per branch
- Attention-based fusion instead of linear

## Current Architecture (Unified)

All variants now live in `mamba_jepa.py`:

```bash
# Standard JEPA (linear input projection)
python mamba_jepa.py --predictor-mlp --epochs 1000 --no-compile

# Single conv branch at stride 4
python mamba_jepa.py --strides 4 --predictor-mlp --epochs 1000 --no-compile

# Multi-scale branches
python mamba_jepa.py --strides 1,4,16 --predictor-mlp --epochs 1000 --no-compile

# Deep supervision
python mamba_jepa.py --per-layer-loss --predictor-mlp --epochs 1000 --no-compile

# Conv branch + deep supervision
python mamba_jepa.py --strides 4 --per-layer-loss --epochs 1000 --no-compile

# Evaluate any checkpoint
python evaluate_representations.py --checkpoint mamba_jepa_model.pt
```

The `--strides` flag controls the encoder:
- **Not set**: Standard `MambaEncoder` (Linear → Mamba layers → LayerNorm)
- **Set**: `MultiScaleEncoder` wrapping `ConvBranch` instances with optional fusion

Old `mamba_jepa_multiscale.py` checkpoints remain loadable through backward compatibility in `evaluate_representations.py`.

## Summary of Best Results

| Model | Data | Epochs | seq_len | Peak Silhouette |
|-------|------|--------|---------|-----------------|
| Supervised next-step | noisy + random walk | 200 | 1024 | 0.435 |
| **JEPA MLP predictor** | **noisy + random walk** | **1000** | **1024** | **0.599** |
| JEPA MLP predictor | noisy + random walk | 1000 | 512 | 0.547 |
| JEPA MLP predictor | noisy + random walk | 3000 | 512 | 0.394 |
| JEPA per-layer loss | noisy + random walk | 1000 | 512 | 0.283 |
| Multi-scale stride=4 | 2× noise, no walk | 1000 | 512 | 0.515 |
| Multi-scale stride=16 | 2× noise, no walk | 1000 | 512 | negative |
| Multi-scale [1,4,16] | 2× noise, no walk | 1000 | 512 | negative |

## Key Takeaways

1. **JEPA > supervised** for representation quality on this task
2. **Weak predictor > strong predictor** — MLP beats Mamba predictor despite higher JEPA loss
3. **Longer context helps** — seq_len=1024 >> 512 >> 128
4. **~1000 epochs is the sweet spot** — overtaining degrades later-layer representations
5. **Moderate downsampling (stride 4) works** — forces slow-structure learning
6. **Extreme downsampling (stride 16) fails** — too much information lost
7. **Multi-branch fusion is tricky** — fast branch dominates with single-step loss
8. **Deep supervision helps stability** but hurts peak performance
9. **LayerNorm is essential** for JEPA training stability (both final and per-layer)
